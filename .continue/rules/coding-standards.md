# Standards de code (Dart/Flutter)

- Null-safety stricte ; types explicites si ambigu.
- Nommage : `PascalCase` (classes), `camelCase` (méthodes/vars).
- Erreurs typées (sealed classes / Either / Result) vers la UI.
- Tests unitaires au moins pour les UseCases modifiés.
- Listes/images : tailles déterministes, cache, pas de rebuilds inutiles.
